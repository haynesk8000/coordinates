Update the application with the following modifications to the **Component Breakdown** and **Velocity Equations** sections.

## 1. Update the Component Breakdown Section

### Initial Velocity Row

The **Initial Velocity** row should always display the projectile's initial velocity components and **must not change** as the simulation time advances.

Replace the current values with the following symbolic component notation:

* **X Component:** **vₓ₀**
* **Y Component:** **vᵧ₀**

These values should remain constant throughout the simulation.

---

### Current Velocity Row

Update the **Current Velocity** row so that it reflects the projectile's velocity at the current simulation time.

#### X Component

* Always display **vₓ₀**.
* Since there is no horizontal acceleration, the horizontal velocity remains constant.

#### Y Component

* Display the symbolic equation:

  * **vᵧ = vᵧ₀ − gt**
* Substitute the **current simulation time** for **t**, while leaving all other variables symbolic.
* For example, if the current time is 2.3 s, display:

  * **vᵧ = vᵧ₀ − g(2.3)**
* Do **not** evaluate the expression numerically.
* If the current vertical velocity is exactly zero (or within the application's tolerance for zero), simply display:

  * **0**

## 2. Update the Velocity Equations Section

Modify the **Velocity Equations** section so that it behaves the same way as the existing **Position Equations** section.

### Requirements

* Update the displayed equations whenever the **current simulation time** changes.
* Use **vₓ₀** and **vᵧ₀** instead of **v₀** in all symbolic expressions.

The equations should be displayed as:

* **Horizontal Velocity**

  * **vₓ = vₓ₀**

* **Vertical Velocity**

  * **vᵧ = vᵧ₀ − gt**

When the simulation time changes:

* Substitute the current value of **t** into the displayed equations while leaving the remaining variables symbolic.
* Follow the same formatting, update behavior, and substitution style used by the existing **Position Equations** section.

Maintain consistent mathematical formatting, typography, spacing, and styling throughout the application.
