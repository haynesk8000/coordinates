Here's a clearer, implementation-ready version for an AI coding agent:

Update the projectile visualization with the following changes to the vector display.

## 1. Replace the Velocity Vectors

Remove the current **v₀** and **g** vectors that are attached to the projectile.

Replace them with separate **vₓ** and **vᵧ** vectors.

---

## 2. Horizontal Velocity Vector (vₓ)

The horizontal velocity vector should represent the projectile's horizontal velocity component.

Requirements:

* Label the vector **vₓ**.
* The vector should always point in the positive x-direction.
* The vector's length should remain constant throughout the simulation since the horizontal velocity does not change.
* Keep the vector anchored to the projectile.

---

## 3. Vertical Velocity Vector (vᵧ)

The vertical velocity vector should represent the projectile's instantaneous vertical velocity.

Requirements:

* Label the vector **vᵧ**.
* Keep the vector anchored to the projectile.
* The vector's length should be proportional to the magnitude of the current vertical velocity.
* The vector's direction should indicate the sign of the vertical velocity:

  * Point upward when **vᵧ > 0**.
  * Have zero length (or be hidden) when **vᵧ = 0**.
  * Point downward when **vᵧ < 0**.
* Update both the length and direction continuously as the simulation advances.

---

## 4. Gravity Vector

Remove the gravity vector from the projectile.

Instead, add a fixed gravity reference vector to the visualization.

Requirements:

* Place the gravity vector near the **far left side** of the display window.
* The vector should always point vertically downward.
* Label the vector **g**.
* The gravity vector should remain fixed in position and should not move with the projectile.
* Its purpose is to serve as a constant visual reference for the direction of gravitational acceleration.

---

## 5. Styling

* Use the same arrow style, line thickness, colors, and typography as the existing vector graphics.
* Ensure the vectors remain clearly visible and do not interfere with the projectile, trajectory, measurement annotations, or other UI elements.
* Update the **vᵧ** vector smoothly during animation to accurately reflect the current vertical velocity at every simulation step.
