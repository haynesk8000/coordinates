### Feature Request: Coordinate Systems Explorer Walkthroughs

#### Objective

Enhance the **Coordinate Systems → Explore (Level 2)** tab by adding guided walkthroughs that systematically teach the concepts of **Translation** and **Rotation** through interactive, validated tasks.

### UI Changes

1. Add a new section immediately above the interactive display window.
2. Create a framed group titled **Walkthroughs**.
3. Inside the frame, place two buttons horizontally:

   * **Translation**
   * **Rotation**
4. Display a progress bar beneath each button.

   * The progress bar represents completion of the ten walkthrough tasks.
   * When all ten tasks are complete, display a distinct visual completion indicator (for example: checkmark, gold border, "Complete" badge, or similar).

---

### Translation Walkthrough

When the **Translation** button is clicked:

1. Open a walkthrough pop-up window.
2. Display the first incomplete translation task from `explorer_tasks.md`.

   * If no tasks have been completed, begin with Task 1.
   * If tasks have already been completed, resume at the next incomplete task.
3. Each task must be displayed in the same order it appears in `explorer_tasks.md`.

Each walkthrough page must display:

* Title
* Description
* Observation
* Explanation

Enhance the Observation and/or Explanation sections with small diagrams or illustrations that help students understand the concept. These visuals should supplement the text from `explorer_tasks.md`, not replace it.

---

### Navigation

Each walkthrough page includes:

* **Previous** button
* **Next** button

Behavior:

* Previous navigates to the previous task.
* Next remains disabled until the current task has been successfully completed.

---

### Task Validation

The application must monitor the interactive workspace and determine whether the student has correctly completed the current task.

Only enable the **Next** button after the required task conditions have been satisfied.

---

### Assistance

If the student has not successfully completed the task within **15 seconds**:

1. Display a helpful hint describing exactly what still needs to be done.
2. The hint should explain the missing or incorrect action required to satisfy the task conditions.
3. Continue monitoring until the task is completed.

---

### Progress Persistence

Persist walkthrough progress between sessions.

When the Translation walkthrough is reopened:

* Resume from the first incomplete task.
* Previously completed tasks remain completed.
* Completed tasks should allow immediate navigation using the enabled **Next** button.

---

### Rotation Walkthrough

Implement identical functionality for the **Rotation** button.

The only difference is that it loads the rotation tasks from `explorer_tasks.md` while preserving:

* ordering
* validation
* progress tracking
* persistence
* hints
* navigation
* diagrams

---

### Progress Indicators

Maintain independent progress for:

* Translation walkthrough
* Rotation walkthrough

Each progress bar should:

* display completion percentage
* update immediately after each completed task
* clearly indicate when all ten tasks are finished

---

### Overall Goal

The walkthrough system should function as an interactive tutor that guides students through the essential concepts of **Translation** and **Rotation** in a structured sequence. Students should only advance after successfully demonstrating each concept in the interactive workspace, ensuring active learning rather than passive reading.
