# Code Review

## Findings

### High - Explore mode does not implement adjustable projectile parameters

`src/App.tsx:31` binds `params` directly to `defaultParameters`, and the Explore UI only exposes preset, rotation, flip, reset, label, and time controls in `src/components/ExploreMode.tsx:109-163`. There are no sliders or state setters for `H`, `h`, `d1`, `d2`, `v0`, or `g`.

Impact: A core acceptance requirement is missing. Students cannot test how the equations and scene respond to changed physical quantities, and the app cannot satisfy "Parameter Controls" for the sandbox.

Recommendation: Store `ProjectileParameters` in React state, add a `ParameterControls` component with sliders for all six quantities, and make sure preset origins recompute or intentionally preserve their world coordinates when parameters change.

### High - Quiz choices always put the correct answer first

`src/quiz/questionBank.ts:8-10` builds choices as `[answer, ...wrongChoices]`, `src/quiz/questionBank.ts:25` stores that order, and `src/components/QuizMode.tsx:56-59` renders the array unchanged.

Impact: Every multiple-choice, equation-selection, sign, origin, and swap question gives away the answer by position. This substantially weakens Quiz mode as an assessment tool even though the question selection itself is randomized.

Recommendation: Shuffle choices per question render or pre-shuffle when selecting questions, while keeping the `answer` field unchanged for grading.

### High - Component quiz questions mark unsimplified or incorrect equations as correct

The "Default origin at the cliff top" component prompt asks for "simplified equations" but marks `x0 + v0 t` and `y0 - 1/2 g t^2` correct in `src/quiz/questionBank.ts:459-465`. The formatter and tests correctly expect the simplified default equations to be `x(t) = v0 t` and `y(t) = -1/2 g t^2`. The ground-origin component question similarly marks `x0 + v0 t` and `y0 - 1/2 g t^2` correct in `src/quiz/questionBank.ts:469-475`, while the app's simplified output is `x(t) = v0 t` and `y(t) = H - 1/2 g t^2`.

Impact: Quiz mode can teach students the opposite of the app's own equation panel and unit tests. This is especially risky because these questions target equation construction, the central learning goal.

Recommendation: Either change the prompts to ask for the general component form, or update the answers and choices to match the simplified substituted equations generated by `formatEquationSet`.

### Medium - Equation panel omits the required general form

`src/components/EquationPanel.tsx:26-39` renders only `simplifiedLatex` for each axis, even though `AxisEquation.generalLatex` is generated by the formatter.

Impact: The UI skips the required bridge from `q(t) = q0 + vq0 t + 1/2 aq t^2` to the simplified substituted form. That makes the displayed equations less traceable to component reasoning.

Recommendation: Render both general equations and simplified equations, with clear labels such as "General component form" and "After substitution."

### Medium - Time slider does not display current physical and coordinate values

`src/components/TimeSlider.tsx:18-30` shows only the current time. `src/components/SceneCanvas.tsx:62` computes the projectile world position for drawing, and `coordinateAtTime` exists in `src/physics/coordinateSystem.ts:124-134`, but Explore mode does not display the current world position or selected-coordinate values.

Impact: A required Explore feature is missing, and students lose a useful check that the same physical point has different coordinate values under different systems.

Recommendation: Add a small live readout near the time slider with `X_world`, `Y_world`, and `{label1}(t)`, `{label2}(t)` values.

### Medium - Custom labels and arbitrary axis swapping are not wired into the UI

The label UI is limited to fixed label sets in `src/components/ExploreMode.tsx:31-35` and buttons in `src/components/ExploreMode.tsx:156-163`. The engine has `swapCoordinateVariables` in `src/physics/coordinateSystem.ts:116-122`, but Explore mode never exposes it.

Impact: Students can select preset #10 for one swapped example, but they cannot swap the current coordinate variables or enter custom labels as required. This limits the relabeling lesson and makes the unused engine function easy to drift out of correctness.

Recommendation: Add two text inputs for custom labels and a swap button/toggle that calls `swapCoordinateVariables(system)`.

### Medium - Interactive SVG controls are not fully keyboard or screen-reader accessible

The axis rotation hit zones and handles are pointer-only and `aria-hidden` in `src/components/SceneCanvas.tsx:264-290`. The origin handle is keyboard movable, but it is exposed as `role="slider"` without `aria-valuemin`, `aria-valuemax`, or `aria-valuenow` in `src/components/SceneCanvas.tsx:302-310`. The parent SVG uses `role="application"` in `src/components/SceneCanvas.tsx:195-200`, which raises the bar for complete keyboard interaction semantics.

Impact: Keyboard and assistive-technology users do not get equivalent access to diagram manipulation. The separate rotation range input helps, but the diagram controls themselves do not meet the stated accessibility requirement.

Recommendation: Either expose the diagram as an image plus conventional external controls, or make each interactive handle a properly named keyboard-operable control with complete ARIA value metadata.

### Medium - Drag-and-drop component questions are implemented as click-to-fill

`src/components/QuizMode.tsx:65-105` implements component questions with buttons that append selected text into blanks. There are no drag sources, drop targets, keyboard drag alternatives, or reorder controls.

Impact: The implemented interaction does not match the required "Drag-and-Drop Components" quiz type. It is usable, but it is not the requested question type and has weaker affordance for placing terms into specific blanks.

Recommendation: Add real drag/drop behavior with accessible keyboard placement, or rename the requirement and UI to "component builder" if click-to-fill is the intended accessible alternative.

### Low - Short-answer grading is brittle

`src/components/QuizMode.tsx:168-172` grades short explanations by exact normalized substring matches against a small list of key ideas. For example, "rotation" does not match the key idea "rotated", and a good answer using "rates" instead of "velocity and acceleration" can be marked wrong.

Impact: Students may receive negative feedback for conceptually correct explanations.

Recommendation: Use a larger synonym/stem list per key idea, or grade by matching groups of acceptable phrases rather than raw substrings.

### Low - Saved quiz score shape is not validated before use

`src/components/QuizMode.tsx:36-43` casts parsed local storage directly to `QuizScore`, and `src/quiz/scoring.ts:26-32` assumes every skill key exists. Invalid JSON is handled, but stale or partially valid JSON can still throw when a missing skill is updated.

Impact: A browser with old or manually edited local storage can crash Quiz mode on answer submission.

Recommendation: Normalize loaded scores by merging them into `emptyScore()` and clamping unexpected fields.

### Low - Build emits a large chunk warning

`npm run build` succeeds, but Vite warns that the main JavaScript chunk is just over 500 kB after minification.

Impact: This is not blocking for a classroom app, but it is a performance watch item, especially on student Chromebooks.

Recommendation: Consider lazy-loading Quiz/Explain mode or splitting KaTeX if first-load performance becomes noticeable.

## Coverage Notes

- The core vector math, coordinate projection, preset equations, rotated symbolic formatting, and scene geometry have useful unit coverage.
- Existing E2E tests cover preset selection, axis flipping, preset-based origin movement, and axis drag rotation.
- There is no test coverage for actual free dragging of the origin, parameter controls, custom labels, arbitrary swapping, local-storage migration, or answer-choice randomization.

## Open Questions

- I did not verify the numbered presets against the original hand-drawn homework figure in `homework.pdf`; I reviewed them against the conceptual preset list in the project instructions.
