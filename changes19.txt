## Reconfigure Application Navigation

### Objective

Restructure the application's navigation hierarchy without changing any existing functionality or content.

### New Navigation Structure

The application should use a two-level tab structure.

#### Top-Level Tabs

Replace the current top-level navigation with the following tabs:

1. Coordinate Systems
2. Projectile Motion
3. Motion Diagrams
4. Relative Motion
5. Uniform Circular Motion

### Coordinate Systems Tab

Move the existing application into the **Coordinate Systems** top-level tab.

Within this tab, retain the current three learning modes as **second-level tabs**:

* Explore
* Explain
* Quiz

Do **not** modify the content or functionality of these modes. Only relocate them into the new navigation hierarchy.

### New Placeholder Modules

Create the remaining four top-level tabs as placeholder modules:

* Projectile Motion
* Motion Diagrams
* Relative Motion
* Uniform Circular Motion

Each placeholder module should contain the same three second-level tabs:

* Explore
* Explain
* Quiz

For this phase, these second-level tabs are placeholders only. They do not require any instructional content or functionality beyond displaying an appropriate placeholder message (e.g., "Content coming soon.").

### Implementation Requirements

* Preserve all existing functionality of the current Coordinate Systems module.
* Do not change any business logic, physics engine, state management, or learning content.
* Only reorganize the user interface and navigation hierarchy.
* The current Coordinate Systems implementation should function exactly as it does today after being moved under its new top-level tab.
* Design the navigation so that each top-level module can later be developed independently while maintaining a consistent user experience.
* Reuse the existing second-level tab component where possible to avoid duplicating code.

### Suggested Navigation Layout

```
Coordinate Systems
    ├── Explore
    ├── Explain
    └── Quiz

Projectile Motion
    ├── Explore
    ├── Explain
    └── Quiz

Motion Diagrams
    ├── Explore
    ├── Explain
    └── Quiz

Relative Motion
    ├── Explore
    ├── Explain
    └── Quiz

Uniform Circular Motion
    ├── Explore
    ├── Explain
    └── Quiz
```

### Acceptance Criteria

1. The application displays five top-level tabs.
2. The existing Coordinate Systems application is contained entirely within the **Coordinate Systems** tab.
3. The existing Explore, Explain, and Quiz views appear as second-level tabs under Coordinate Systems.
4. All existing Coordinate Systems functionality remains unchanged.
5. Each of the four new top-level modules contains placeholder Explore, Explain, and Quiz tabs.
6. Placeholder tabs display a simple "Coming Soon" (or equivalent) message without errors.
7. The navigation architecture is modular and ready for future expansion of each physics topic.
