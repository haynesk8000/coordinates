Update the application so the **Initial Velocity**, **Current Velocity**, and **Velocity Equations** displays respond correctly when the coordinate system is rotated.

## 1. Component Breakdown Section

In the **Component Breakdown** section, update both the **Initial Velocity** row and the **Current Velocity** row so their x- and y-components change symbolically when the coordinate system is rotated.

Use the same symbolic transformation style currently used for the **Acceleration** row.

### Initial Velocity Row

* When the coordinate system is not rotated, display:

  * **x component:** vₓ₀
  * **y component:** vᵧ₀

* When the coordinate system is rotated, display the transformed symbolic components using the rotated coordinate system, consistent with the acceleration row’s formatting.

### Current Velocity Row

* When the coordinate system is not rotated, display:

  * **x component:** vₓ₀
  * **y component:** vᵧ₀ − gt

* When the coordinate system is rotated, display the transformed symbolic current velocity components using the rotated coordinate system, consistent with the acceleration row’s formatting.

The displayed values should remain symbolic and should update dynamically whenever the coordinate system rotation changes or the current time changes.

## 2. Velocity Equations Section

Update the **Velocity Equations** section so the velocity equations also reflect the current coordinate system rotation.

* Use **vₓ₀** and **vᵧ₀** as the initial velocity components.

* When the coordinate system is not rotated, display:

  * **vₓ = vₓ₀**
  * **vᵧ = vᵧ₀ − gt**

* When the coordinate system is rotated, display the transformed symbolic velocity equations in the rotated coordinate system.

* Use the same symbolic rotation convention and formatting already used by the position equations and acceleration display.

Ensure all velocity-related symbolic displays update consistently when either the coordinate system rotation or the current simulation time changes.
